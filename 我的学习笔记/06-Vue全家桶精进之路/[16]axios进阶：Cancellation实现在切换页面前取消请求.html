# Cancellation

  • 你可以使用一个取消令牌 cancel token取消请求
  • axios 取消令牌 API 基于撤回的可取消承诺提议。
  • 别人的笔记——参考axios CancelToken 取消频繁发送请求的用法和源码解析 (opens new window)

# 通过 TS 类型文件了解 Cancellation

export interface CancelTokenStatic {
  // ？？？猜测，这里带了static字眼，可能是axios的一个静态属性/方法，这个属性或方法属于这个class，而不是实例
  // 实例这个class时生成CancelToken类型的实例
  new (executor: (cancel: Canceler) => void): CancelToken;
  source(): CancelTokenSource; //source()这个方法的返回值是CancelTokenSource
  // CancelTokenStatic是个类，实例必须是CancelToken
  // CancelTokenStatic这个类底下还有静态属性
}

export interface CancelToken {
  // ？？？这个没有static字眼的CancelToken属于动态属性，会在原型链prototype中能够被访问到，应该就是axios.CancelToken
  promise: Promise<Cancel>; //promise返回的第一个参数res的类型是个Cancel对象
  // 注意：reject无法被约束
  reason?: Cancel;
  throwIfRequested(): void; //方法没有返回值 throw:抛出错误
  // 我要cancel取消这个请求时这个请求已经完成了，这被认为是一种操作错误，抛出这个错误
}

export interface Cancel {
  //CancelToken的reason原因属性值是个对象，{msg:str}
  message: string;
  // Cancel是个对象
}

export interface CancelStatic {
  // reason原因属性值是由axios的静态属性CancelStatic得到的msg
  new (message?: string): Cancel;
  // CancelStatic是个class
}

export interface CancelTokenSource {
  // 执行source()方法的返回值，一个canceltoken实例和一个cancel方法
  token: CancelToken; //实例
  cancel: Canceler; //方法
}

export interface Canceler {
  // 执行source()返回的CancelTokenSource中的cancel方法
  (message?: string): void;
  // Canceler是个函数，接收一个string参数，返回是void空，只执行不return。
}

# 方案一:使用 CancelToken.source 工厂创建 cancel token

const CancelToken = axios.CancelToken; //axios原型链上的一个方法/或实例
const source = CancelToken.source(); //CancelTokenStatic上的.source()方法将会得到一个包含token实例CancelToken和canceler方法的对象CancelTokenSource

axios
  .get("/user/12345", {
    cancelToken: source.token, //source.token就是这个实例CancelToken，给config配置对象属性cancelToken，注意区分大小写
  })
  // 当axios请求完成后走的是then的代码块，如果then代码块中存在错误代码信息，这时就会进入catch中抛出异常
  // catch()方法捕获抛出的异常。
  .catch(function (thrown) {
    //throwIfRequested,cancel取消这个请求时这个请求已经完成了，这被认为是一种操作错误，抛出这个错误
    // axios.isCancel方法接收的是错误参数thrown
    // axios.isCancel方法用来判断异常对象thrown是不是取消原因对象Cancel,即{message: string}，返回true或false。
    // 如果是true,就能读取到thrown.message
    if (axios.isCancel(thrown)) {
      console.log("Request canceled", thrown.message);
    } else {
      // handle error
    }
  });

axios.post(
  "/user/12345",
  {
    name: "new name",
  },
  {
    cancelToken: source.token,
  }
);

// 取消请求 (参数message是可选项)
source.cancel("msg-用户取消了请求");

# 方案二（推荐）:通过将 执行程序函数 传递给 CancelToken 构造函数 来创建 cancel token

  • You can also create a cancel token by passing an executor function to the CancelToken constructor:
const CancelToken = axios.CancelToken;
let cancel; //给一个变量cancel用来存储

axios.get("/user/12345", {
  cancelToken: new CancelToken(function executor(c) {
    // 带有new说明这是CancelTokenStatic，这里的参数c是个Canceler函数，等同于CancelTokenSource中的cancel方法
    // new的结果是CancelToken
    // 将这个cancel方法/Canceler函数/c保存到变量cancel中
    cancel = c;
  }),
});

// 现在这个变量cancel存的就是cancel方法/Canceler函数/c（三者指的是同一个方法），然后执行它，请求就取消了
cancel();

# 实际应用

# 对请求进行缓存

  • 声明一个数组或者变量作为缓存池子。更推荐对象。
  • 对所有正在进行中的请求进行缓存。
  • 并给每一个 cancel 一个唯一的标识符来保证每一次取消掉的只有要取消的哪一个请求

# 在什么时候取消请求

  1. 在请求发起前判断缓存列表中该请求是否正在进行，如果有则取消本次请求。
  2. 在任意请求完成后，需要在缓存列表中删除该次请求，以便可以重新发送该请求。
  3. 在切换页面前，将未完成的请求取消。

# 参考他人案例

  • 别人的案例——axios 中取消阻止重复请求（防抖请求） (opens new window)
  • 如下，参考参考。
// 正在进行中的请求列表
let reqList = [];

/**
 * 阻止重复请求
 * @param {array} reqList - 请求缓存列表
 * @param {string} url - 当前请求地址
 * @param {function} cancel - 请求中断函数
 * @param {string} errorMessage - 请求中断时需要显示的错误信息
 */
const stopRepeatRequest = function (reqList, url, cancel, errorMessage) {
  const errorMsg = errorMessage || "";
  for (let i = 0; i < reqList.length; i++) {
    if (reqList[i] === url) {
      cancel(errorMsg);
      return;
    }
  }
  reqList.push(url);
};

/**
 * 允许某个请求可以继续进行
 * @param {array} reqList 全部请求列表
 * @param {string} url 请求地址
 */
const allowRequest = function (reqList, url) {
  for (let i = 0; i < reqList.length; i++) {
    if (reqList[i] === url) {
      reqList.splice(i, 1);
      break;
    }
  }
};

const service = axios.create();

// 请求拦截器
service.interceptors.request.use(
  (config) => {
    let cancel;
    // 设置cancelToken对象
    config.cancelToken = new axios.CancelToken(function (c) {
      cancel = c;
    });
    // 阻止重复请求。当上个请求未完成时，相同的请求不会进行
    stopRepeatRequest(reqList, config.url, cancel, `${config.url} 请求被中断`);
    return config;
  },
  (err) => Promise.reject(err)
);

// 响应拦截器
service.interceptors.response.use(
  (response) => {
    // 增加延迟，相同请求不得在短时间内重复发送
    setTimeout(() => {
      allowRequest(reqList, response.config.url);
    }, 1000);
    // ...请求成功后的后续操作
    // successHandler(response)
  },
  (error) => {
    if (axios.isCancel(thrown)) {
      console.log(thrown.message);
    } else {
      // 增加延迟，相同请求不得在短时间内重复发送
      setTimeout(() => {
        allowRequest(reqList, error.config.url);
      }, 1000);
    }
    // ...请求失败后的后续操作
    // errorHandler(error)
  }
);

# 自己实现一个并记录（初级版本）

  • 配置：cancelToken 是设置在请求的配置 config 对象里的，实例 axios 时配置
  • 实例的时候还没有发出请求，此时不应该将 cancel 放入池子
  • 收集 cancel 放入池子，放入池子中时要确认上一次该请求是否已经完成或者 cancel,所以放在请求前，在拦截机设置
  • 而执行 cancel 方法并不是在创建请求实例的时候，而是在发送请求之前/切换页面之前/任意想要取消请求的时候
  • 这里需要区分两种情况
  • 一种是我要取消正在进行的请求(使用 cancel)，一种是我为了防抖，发现已经在请求了我就不重新发送请求(使用 loading 状态机进行管理)
  • 因此，思路应该是：
  • 1.节流
  • 每次发送请求前判断这个请求是否正在请求（状态机 loading)，如果正在请求,我就不发，避免重复请求。
  • 等到上一个请求结束了（响应完成），会自动 cancel，这时才可以发送新的同一个请求
  • 2.请求前
  • 确定这个请求没有正在请求，我就可以发送请求了
  • 但是在我发送请求前，保险操作我得确保上一个请求已经 cancel 掉(如果 cancel 还在手动 cancel)，并在池子中保存这个请求的 cancel
  • 3.响应后
  • 请求完成自动 cancel()了，我只需要将池子中的 cancel 移除
  • 两个位置：拦截响应/请求成功 promise 成功接口,后者位置更佳，能确保数据已经获取到了。
  • 4.切换页面前，cancel()全部，清空池子
  • 5.特殊情况，cancel()指定项并在池子中移除
// import Vue from "vue";
import Axios from "axios";

//设置不变的请求config
export const AXIOS_OPTION = {
  baseURL: "",
  headers: {
    "Content-Type": "application/json",
  },
  withCredentials: true, //请求时加上cookie
  timeout: 30000, //超时时间，
};

//todo:请求不存在才实例并发送请求,添加状态机loading

export class createAxios {
  //类实例将运行constructor然后将实例结果保存给_axiosInstance
  _axiosInstance = null;
  //请求的config
  _axiosRequestConfig = {};
  // 拿到CancelToken函数
  CancelToken = Axios.CancelToken;

  //cancel池子，动态属性，所有的实例都可以网这个池子添加要收集的cancel
  static cancelObj = {};
  //constructor本质上相当于MVC中C里面的init初始化
  constructor(
    url,
    options = {
      baseURL: "",
      headers: {
        "Content-Type": "application/json",
      },
      withCredentials: true, //请求时加上cookie
      timeout: 30000, //超时时间，
    }
  ) {
    this._axiosRequestConfig = options;
    this._axiosInstance = Axios.create(this._axiosRequestConfig);
    this.setInterceptor(this._axiosInstance);
    // this.httpRequest = this.httpRequest.bind(this);
  }

  setInterceptor(instance) {
    //发送请求之前
    instance.interceptors.request.use((config) => {
      console.log("interceptors.request.");
      const cancelKey = this.getPendingUrl(config);
      //判断config是否已配置cancelToken
      config.cancelToken = new this.CancelToken((c) => {
        //判断请求是否存在,存在就让cancel停止请求，添加本次请求到池子中
        if (createAxios.cancelObj[cancelKey]) {
          this.removeCancel(cancelKey);
        }
        this.addCancel(cancelKey, c);
      });
      return config;
    });
    // 添加响应后拦截器，处理response
    instance.interceptors.response.use(
      (res) => {
        const cancelKey = this.getPendingUrl(res.config);
        this.removeCancel(cancelKey);
        return res;
      },
      (error) => {
        if (Axios.isCancel(error)) {
          //判断是否是由于手动cancel引起的中断的报错，是的话就不做处理
          return Promise.reject("重复请求，请求被取消");
        } else {
          console.log(error);
          return error;
        }
      }
    );
  }

  getPendingUrl({ method, url }) {
    //设置标识cancel的key
    return [method, url].join("&");
  }
  addCancel(cancelKey, cancel) {
    createAxios.cancelObj[cancelKey] = cancel;
  }
  removeCancel(cancelKey) {
    //请求重复或响应完成时，取消该请求并在池子中移除
    const cancel = createAxios.cancelObj[cancelKey];
    cancel && cancel();
    delete createAxios.cancelObj[cancelKey];
  }
  canceler(cancel) {
    // 取消当前请求，已在请求中了，请求防抖
    setTimeout(cancel, 1);
  }
  removeAllCancel() {
    // 监听路由变化确认页面切换，切换页面前调用将现存所有进行中的请求cancel
    Object.keys(createAxios.cancelObj).forEach((cancelKey) => {
      createAxios.cancelObj[cancelKey]();
    });
    createAxios.cancelObj = {};
  }

  //todo:这里还可以加一个loading=false,默认不加loading，可选择是否有loading效果
  httpRequest = (url, data = "", method = "get", errorTitle = "") => {
    // console.log(this);
    // console.log(this._axiosInstance);
    // console.log(url);
    // console.log(method);
    console.log(url);
    return new Promise((resolve, reject) => {
      this._axiosInstance
        .request({
          url,
          method,
          params: method === "get" ? data : {},
          data: method !== "get" ? data : {},
        })
        .then((res) => {
          resolve(res.data);
        })
        .catch((err) => {
          // 出错时打印错误
          const tip = "请求" + errorTitle + "失败";
          console.log(tip);
          reject(err);
        })
        .finally();
    });
  };

  // checkStatus(status, errno) {
  // return !status && [4100, 1].includes(errno);
  // }
}

export const vueAxios = new createAxios(AXIOS_OPTION);
export const httpRequest = vueAxios.httpRequest;
