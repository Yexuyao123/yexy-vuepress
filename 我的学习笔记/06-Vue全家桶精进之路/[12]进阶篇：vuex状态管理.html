# Axios 请求

  • 关联博客——[01]衔接篇：Axios 代替 ajax

  • axios 的简介：axios 是一个基于 Promise 用于浏览器和 nodejs 的 HTTP 客户端，它本身具有以下特征：

    • 从浏览器中创建 XMLHttpRequest
    • 从 node.js 发出 http 请求
    • 支持 Promise API
    • 拦截请求和响应
    • 转换请求和响应数据
    • 取消请求
    • 自动转换 JSON 数据
    • 客户端支持防止 CSRF/XSRF

# Axios 中的 get 请求

  • 安装npm i axios
  • 引入加载import axios from "axios"
  • 将 axios 全局挂载到 vue 原型上Vue.prototype.$http = axios 之后就可以通过 this.$http 调用 axios
  • 发出请求axios.get().then().catch()
<template>
  <div class="hello">
    <h1>我是axiosAPP,用来发送请求拦截响应</h1>
    <button @click="getData">发送请求</button>
  </div>
</template>

<script>
import axios from "axios";
import Vue from "vue";
Vue.prototype.$http = axios;

export default {
  name: "Study",
  methods: {
    getData() {
      this.$http
        .get("https://cnodejs.org/api/v1/topics", {
          params: {
            page: 1,
            limit: 10,
          },
        })
        .then(function (res) {
          // 成功时
          // 打印axios的结果
          console.log(res);
          // 打印http响应
          console.log(res.data);
          // 打印响应的数据，由后端设定，可以是res.data.result
          console.log(res.data.data);
        })
        .catch(function (err) {
          // 出错时打印错误
          console.log(err);
        });
    },
  },
};
</script>

# Axios 中的 post 请求

  • POST 请求传递数据有两种方式
  • 方式一：form-data
  • 服务器/后端获取到的是?name=ari&age=18查询参数的形式
  • 但是请求的时候数据不是被携带在 url 里，在 url 里数据就直接暴露出来了，因此不能用 params 对象
  • 在 axios 中，POST 请求接收的参数必须是 form-data
  • qs 插件——qs.stringify
  • npm i qs安装
  • 方式二：x-www-form-urlencoded{page:1,limit:10}
<script>
// qs的使用
import axios from "axios";
import Vue from "vue";
import qs from "qs";
Vue.prototype.$http = axios;

export default {
  name: "Study",
  methods: {
    postData() {
      this.$http.post(
        "https://linzhihui.online/",
        qs.stringify({
          page: 1,
          limit: 10,
        })
      );
    },
  },
};
</script>

# vuex 的 store 用法 this.$store.xxx

  • 功能：用来管理状态，共享数据，在各个组件之间管理外部状态
  • 应用：比如电商的用户登陆状态，淘宝购物车状态
  • 如何使用？
  1. npm i vuex安装
  2. main.js 中import Vuex from "vuex";引入 vuex，并通过 use 方法使用它Vue.use(Vuex);
  3. 创建状态仓库
  4. 在 Vue 实例对象中注册 store，并设置全局状态 state
// main.js中
import Vuex from "vuex";
Vue.use(Vuex);
// 创建状态仓库
// .Store()中首字母必须大写
// 全局状态state属性不可更名
const store = new Vuex.Store({
  state: {
    num: 888,
  },
  mutations: {},
  actions: {},
  modules: {},
});
new Vue({
  router,
  // 注册store
  store,
  render: (h) => h(App),
}).$mount("#app");

# mutation 的提交荷载（Payload）

const store = new Vuex.Store({
  //全局状态
  state: {
    num: 666,
    commitName: null,
  },
  //变化、定义改变状态的方法
  mutations: {
    increase(state, params) {
      state.num = state.num + params.addNum;
      state.commitName = params.name;
    },
  },
  actions: {},
  modules: {},
});

# mutation 需遵守 Vue 的响应规则
  • mutation 必须是同步函数
  • 最好在 store 中提前初始化所需属性
  • 当需要在 state 对象上添加属性时，有下列两种方案：
  1. 使用Vue.set(obj,"newProp",123)
  2. 以新对象替换老对象，如：利用 stage-3 的对象展开运算符state.obj={...state.obj,"newProp":123}

# 方法二：使用 actions 定义方法：this.$store.dispatch("actions 定义的方法")

  • 注意：actions 提交的是 mutations，而不是直接变更 status 状态
  • action 字面意思就是行动行为，可以理解为对存储函数 mutation 的动作进行修饰，添加时间轴等等。
  • actions 可以包含异步操作，但 mutations 只能包含同步操作
const store = new Vuex.Store({
  //全局状态
  state: {
    num: 666,
  },
  //变化、定义改变状态的方法
  mutations: {
    decrease(state) {
      console.log(this); //返回store对象
      state.num--;
    },
  },
  actions: {
    //context为上下文对象
    // 在actions中通过 context.commit("decrease") 调用mutations定义的方法
    decreaseAction: function (context) {
      console.log(context); // 返回getters对象
      setTimeout(() => {
        context.commit("decrease");
      }, 1000);
    },
  },
  modules: {},
});

# vuex 状态管理的流程

  • view —— methods —— actions —— mutations —— state —— getters —— computed —— view
  • 直接操纵 state 的是 mutations，actions 不是必须的。
  • 方法获取 state 的本质都是通过 getters 来获取的，当没有定义 getters 是，则用默认 getters 方法
  • 养成习惯手写 getters

# getters 选项

  • 场景：当前状态 state 为余额 10 元，消费 20 块钱，使用 actions — mutations 对 state 操作，余额变为-10 元，显然很不合理
  • 这就需要对操作做一些预判处理
  • 下面举个简单的栗子，当余额不足时，使之为 0
new Vuex.Store({
  //全局状态
  state: {
    num: 666,
  },
  mutations: {
    // 通过触发函数方法，commit保存传递的数据,调用mutations方法
    decrease(state) {
      state.num = state.num - 500;
    },
  },
  actions: {},
  getters: {
    // 通过computed计算属性访问state保存的数据，调用getters函数过滤数据
    getNum(state, param) {
      console.log("传递的参数是：" + param);
      return state.num > 0 ? state.num : 0;
    },
  },
  modules: {},
});

<template>
  <div>
    <p>正常拿到的数据 余额是{{ getVuexNum }}</p>
    <p>设置getters(余额不足时为0)后拿到的数据 余额是{{ balanceToZero }}</p>
    <button @click="minus">使用mutations改变store状态按钮 减500元<button>
  </div>
</template>

<script>
export default {
  name: "vuexStudy",
  computed: {
    // 访问 或 获取 到state中的数据
    // 方法1.默认getter访问state
    getVuexNum: function() {
      return this.$store.state.num;
    },
    // 方法2.设置getters方法后，this.$store.getters.xxx访问state
    balanceToZero: function() {
      return this.$store.getters.getNum;
    }
  },
  methods: {
    // 通过监听函数触发方法，并提交要传递的数据param
    minus() {
      // 调用store中的mutations函数，提交传递的参数param
      this.$store.commit("decrease",param);
    },
  }
};
</script>

# Module 模块化

// 声明与实例化
const moduleA = {
  state: {},
  mutations: {},
  actions: {},
  getters: {},
};
const moduleB = {
  state: {},
  mutations: {},
  actions: {},
  getters: {},
};

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB,
  },
});

// 调用
store.state.a; //moduleA的state状态
store.state.b; //moduleB的state状态
