# 构建 DOM 节点树的方法(为什么使用 render 函数)

  • 高效地更新节点比较困难，但是你无需手动去更新节点。
  • 你只需要告诉 Vue 你希望页面上的 HTML 是什么，这可以是在一个模板里,或者一个渲染函数里。
  • 在这两种情况下，Vue 都会自动保持页面的更新，即便 blogTitle 发生了改变。

# 方案一：使用 Vue 的 template 模板定义组件

<!-- 案例：插入文本 自由切换 不同标题等级 -->
<!-- 方案一：使用Vue的template模板定义组件 -->
<body>
  <div id="app">
    <child :level="level"> 我是蜡笔小新 </child>
  </div>

  <!-- 注意：template标签里面只允许有一个子节点 -->
  <!-- 缺点1：使用v-if来回切换渲染DOM -->
  <!-- 缺点2：在组件template写入大量插槽，每一个级别的标题中重复书写<slot></slot>，代码冗长 -->
  <template id="hdom">
    <div>
      <h1 v-if="level==1">
        <slot>默认插槽</slot>
      </h1>
      <h2 v-if="level==2">
        <slot>默认插槽</slot>
      </h2>
      <h3 v-if="level==3">
        <slot>默认插槽</slot>
      </h3>
    </div>
  </template>
</body>

<script>
  Vue.component("child", {
    props: ["level"],
    // 使用"#hdom"就可以将<template id="hdom"></template>引入进来
    template: "#hdom",
  });
  const app = new Vue({
    el: "#app",
    data: {
      // 通过level取值123选择将内容放入不同等级的标题插槽中
      level: 1,
    },
  });
</script>

# 方案二：使用 Vue 的 render 函数定义组件

<!-- 案例：插入文本 自由切换 不同标题等级 -->
<!-- 方案二：使用Vue的render函数定义组件 -->
<body>
  <div id="app">
    <child :level="level"> 我是蜡笔小新 </child>
  </div>
</body>

<script>
  Vue.component("child", {
    props: ["level"],
    render: function (createElement) {
      return createElement("h" + this.level, this.$slots.default);
    },
  });
  const app = new Vue({
    el: "#app",
    data: {
      // 通过level取值123选择将内容放入不同等级的标题插槽中
      level: 1,
    },
  });
</script>

# createElement 方法(返回 虚拟 DOM)

Vue 通过建立一个虚拟 DOM 来追踪自己要如何改变真实 DOM。

function(createElement){
  return createElement('h1', this.blogTitle)
}
1
2
3
  • createElement 返回什么？
  • 返回的不是一个实际的 DOM 元素。
  • 返回的是一个节点描述(createNodeDescription)
  • 告诉 Vue 页面上需要渲染什么样的节点，包括及其子节点的描述信息。
  • 返回的是一个虚拟节点(virtual node)，常简写为VNode。

# render 函数 createElement 方法 参数解读

Vue.component("child", {
  props: ["xxx"],
  render: function (createElement) {
    return createElement(参数1, 参数2, 参数3);
  },
});

# createElement 参数 1 (String | Object | Function)

  • 参数 1 必选
  • 参数 1 可以是 String | Object | Function 三者之一
  • String：HTML 标签名
  • Object：含有数据选项的对象
  • Function：方法返回含有数据选项的对象

# createElement 参数 2 (Object)

  • 参数 2 可选
  • 参数 2 是数据对象，只能是 Object
  • 对象可选属性查看官方文档深入数据对象 (opens new window)

# createElement 参数 3 (String | Array)

  • 参数 3 可选
  • 参数 3 可以是 String | Array 二者之一，Array 比较常用
  • 参数 3 作为参数 1 节点的子节点
  • 由于参数 2 一定是对象，所以可以只传参数 1 和参数 3 createElement(参数1,参数3)

# 示例

…{
  render: function (createElement) {
    return createElement(
      // 参数1 (必填项)
      // String | Object | Function
      // HTML标签名|组件选项对象|能返回string或object的async函数。
      // 示例
      // String
      "div",
      // Object
      {template:"<div>哈哈哈哈</div>"}
      // Function
      function(){
        return {template:"<div>哈哈哈哈</div>"}
      }.call

      // 参数2 (可选项)
      // Object
      // 一个与模板中 attribute 对应的数据对象。
      // 示例
      {
        // 与 `v-bind:class` 的 API 相同
        class:{
          foo:true,
          active:false,
        },
        // 与 `v-bind:style` 的 API 相同
        style:{
          color:"red",
          fontSize:"24px",
        },
        // 普通的 HTML attribute
        attrs: {
          id: "foo",
          src:"baidu.com",
        },
        // 组件 prop
        props: {
          myProp: "bar"
        },
        // DOM property
        domProps: {},
        // 事件监听器在 `on` 内，
        on: {},
        // 仅用于组件，用于监听原生事件`vm.$emit` 触发的事件。
        nativeOn: {},
        // 自定义指令。
        directives: [],
        // 等，详见官方文档
      },

      // 参数3 (可选项)
      // String | Array
      // 参数3是 子级虚拟节点 (VNodes)，由 `createElement()` 构建而成，也可以使用字符串来生成“文本虚拟节点”。
      // 示例
      [
        // 使用字符串来生成“文本虚拟节点”
        "先写一些文字",
        // 由 `createElement()` 构建而成
        createElement("h1", "我是h1标题"),
        createElement(MyComponent, {
          props: {
            someProp: "foobar",
          },
        }),
      ]

    );
  },
}

# 插槽 this.$slots 在 render 函数的应用

this.$slots.slotname 拿到 DOM 元素

<!-- 插槽this.$slots 在 render 函数的应用 -->
<body>
  <div id="bpp">
    <my-component>
      <!-- 这两段无需定义，使用slot=default就好了 -->
      <p>文章第一段</p>
      <p>文章第二段</p>
      <!-- 定义插槽slot="slotname" -->
      <h3 slot="header">标题</h3>
      <h5 slot="footer">文章结尾</h5>
    </my-component>
  </div>
</body>
<script>
  Vue.component("my-component", {
    render: function (createElement) {
      // 拿到DOM元素
      const header = this.$slots.header;
      const footer = this.$slots.footer;
      return createElement("div", [
        // 第三个参数是子节点
        // 本身createElement返回的是虚拟节点Vnode，是一种动态更新
        // 只有在数据有变动的节点才重新渲染，避免所有插槽一起来回切换渲染
        createElement("header", header),
        createElement("main", this.$slots.default),
        createElement("footer", footer),
      ]);
    },
  });
  const bpp = new Vue({
    el: "#bpp",
  });
</script>

# 使用 props 在 render 函数传递数据

<body>
  <!-- 需求：点击按钮来回切换照片 -->
  <div id="cpp">
    <button @click="switchShow">点我切换照片</button>{{show}}
    <!-- 将是否要展示照片的数据show传递给子组件 -->
    <show-component :show="show"></show-component>
  </div>
</body>
<script>
  Vue.component("show-component", {
    // 先从组件中接收传递的数据show
    props: ["show"],
    render: function (createElement) {
      // 通过props接收进来的数据在组件中可以通过this.show访问
      let imgsrc = "";
      if (this.show) {
        imgsrc =
          "https://wx4.sinaimg.cn/mw1024/726b7c3egy1gll0mbrb0hj23341e84qq.jpg";
      } else {
        imgsrc =
          "https://wx1.sinaimg.cn/mw1024/726b7c3ely1gobaptlr1mj23341qo7wj.jpg";
      }
      return createElement("img", {
        style: {
          width: "1000px",
          height: "480px",
        },
        attrs: {
          src: imgsrc,
        },
      });
    },
  });
  const cpp = new Vue({
    el: "#cpp",
    data: {
      show: false,
    },
    methods: {
      switchShow: function () {
        this.show = !this.show;
      },
    },
  });
</script>

# v-model 在 render 函数的应用

<body>
  <!-- v-model 在 render 函数的应用 -->
  <!-- 需求：input框输入啥，就在组件中显示啥 -->
  <div id="dpp">
    我输入的名字{{name}}
    <!-- v-on定义一个监听事件input,用showname方法接收从子组件，也就是从render函数中传递过来的数据inputname -->
    <vmodel-component :name="name" @input="showname"> </vmodel-component>
    <!-- 也可使用v-model,无需绑定merhods方法 -->
    <!-- <vmodel-component :name="name" v-model="name"> -->
  </div>
</body>
<script>
  Vue.component("vmodel-component", {
    // 先从组件中接收传递的数据"name"
    props: ["name"],
    render: function (createElement) {
      // 拿到当前的vue组件实例
      const that = this;
      return createElement("input", {
        domProps: {
          value: that.name,
        },
        on: {
          // 监听input事件时，会将原生的默认事件event传递进来
          input: function (event) {
            // console.log(this) //Window{}
            // console.log(that) //Proxy{}
            // 子组件 this.$emit 给父组件传递数据
            // (第一个参数是事件名,后面的参数是要传递的数据)
            // 这个数据就是在input框输入的数据event.target.value
            that.$emit("input", event.target.value);
          },
        },
      });
    },
  });
  const dpp = new Vue({
    el: "#dpp",
    data: {
      name: "Jack",
    },
    methods: {
      // 在自定义事件中用一个参数来接收 子组件传递过来的数据
      showname: function (inputname) {
        this.name = inputname;
      },
    },
  });
</script>

# 作用域插槽在 render 函数的应用

<body>
  <!-- 作用域插槽在 render 函数的应用 -->
  <div id="epp">
    <hello-component>
      <!-- 作用域插槽使用scopedSlots，一般要写在template标签中 -->
      <template scope="prop"> {{prop.text}} {{prop.msg}} </template>
    </hello-component>
  </div>
</body>
<script>
  Vue.component("hello-component", {
    // 相当于<div><slot :text=text></slot></div>
    render: function (createElement) {
      return createElement(
        "div",
        // this.$scopedSlots.default()接收一个对象，包含要传递的数据
        this.$scopedSlots.default({
          text: "我是子组件传递过来的数据",
          msg: "呵呵",
        })
      );
    },
  });
  const epp = new Vue({
    el: "#epp",
  });
</script>

# 函数化组件(context 对象)

  • functional:true：表示当前的 vue 实例无状态，无实例。
  • 无实例表示这个组件内部没有 this 这个概念，传递过来的数据如何拿到？
  • vue 提供了第二个参数 context，上下文对象，通过 context 对象拿到数据。
  • 包括 data、props、slots、parent、children 都可通过 context 拿到
  • 说明 context 可以拿到父组件的内容，也可以拿到子组件的内容
<body>
  <!-- 函数化组件 -->
  <div id="fpp">
    <fun-component value="lalala"> </fun-component>
  </div>
</body>
<script>
  Vue.component("fun-component", {
    functional: true, //表示当前的vue实例无状态，无实例
    props: ["value"],
    // vue 提供了第二个参数 context，上下文对象，通过 context 对象拿到数据。包括 data、props、slots、parent、children 都可通过 context 拿到
    render: function (createElement, context) {
      return createElement("button", {
        domProps: {
          innerHTML: "点我点我打印context对象",
        },
        on: {
          // 点击按钮时
          click: function () {
            console.log(context);
            console.log(context.parent);
            console.log(context.parent.msg);
            console.log(context.props.value);
          },
        },
      });
    },
  });
  const fpp = new Vue({
    el: "#fpp",
    msg: "fpp的数据",
    //可在context.parent.options.msg查到
    // 通过context.parent.msg可读取到
  });
</script>
