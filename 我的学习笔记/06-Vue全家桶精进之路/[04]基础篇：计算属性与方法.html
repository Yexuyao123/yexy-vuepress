# Vue.js 计算属性

# 什么是计算属性

当文本插值表达式过长，逻辑更为复杂时，就会变得十分臃肿，难以阅读和维护。 所有的计算属性都以函数的形式写在 vue 实例内的computed选项内，最终返回计算后的结果。

# 计算属性用法

  1. 一个计算属性里可以完成各种复杂的逻辑，包括运算、函数调用等，只要最终返回一个结果就可以
  2. 计算属性还可以依赖多个 Vue 实例的数据，只要其中任一数据发生变化，计算属性就会重新执行，视图也会更新。
  3. 如果计算属性直接跟一个 function 函数，默认就是 getter 函数，每个计算属性包含一个 getter 和一个 setter

# 举个栗子 1

<div id="dataApp">{{reversedText}}</div>
<!-- 需求：将app.text中的123，456，789转成789，456，123 -->
<script>
  let app = new Vue({
    el: "#dataApp",
    data: {
      text: "123,456,789",
    },
    computed: {
      reversedText: function () {
        return this.text.split(",").reverse().join(",");
      },
    },
  });
</script>

<span>第一个购物车应收{{spend2}}元</span><br />
<!-- 需求：计算购物车中的价格 -->
<script>
  let app = new Vue({
    el: "#dataApp",
    data: {
      shoppingCard1: [
        {
          name: "iphone7Plus",
          price: 6999,
          count: 2,
          for: "yyyexy",
        },
        {
          name: "ipad",
          price: 3999,
          count: 5,
          for: "Linzhihui",
        },
      ],
      shoppingCard2: [
        {
          name: "dongfangRedTea",
          price: 4.5,
          count: 233,
          for: "yyyexy",
        },
        {
          name: "bashiMilk",
          price: 3.5,
          count: 20,
          for: "Linzhihui",
        },
      ],
    },
    computed: {
      spend1: function () {
        let spend = 0;
        for (let i = 0; i < this.shoppingCard1.length; i++) {
          spend = +this.shoppingCard1[i].price * this.shoppingCard1[i].count;
        }
        return spend;
      },
      spend2: function () {
        let spend = 0;
        for (let i = 0; i < this.shoppingCard2.length; i++) {
          spend += this.shoppingCard2[i].price * this.shoppingCard2[i].count;
        }
        return spend;
      },
    },
  });
</script>

# 举个栗子 4 getter 与 setter

计算属性默认只有 getter，不过在需要时你也可以提供一个 setter

computed: {
  fullName: {
    // getter
    get: function () {
      return this.firstName + ' ' + this.lastName
    },
    // setter
    set: function (newValue) {
      var names = newValue.split(' ')
      this.firstName = names[0]
      this.lastName = names[names.length - 1]
    }
  }
}

computed 的东西是不允许直接赋值行为的，它的值永远只会是 get 函数计算出来的，赋值行为只会触发 set 函数。

# 计算属性缓存 vs 方法

在表达式中调用计算属性和调用方法可以实现相同的效果。

# 计算属性(有缓存)

基于它们的响应式依赖进行缓存的。只在相关响应式依赖发生改变时它们才会重新求值。 这就意味着只要 data 选项中的 msg 还没有发生改变，多次访问计算属性会立即返回之前的计算结果，而不必再次执行函数。 不管渲染不渲染，只要计算属性依赖的数据未发生变化，就永远不变。

<p>{{ now }}</p>
1
// 下面的计算属性将不再更新，因为 Date.now() 不是响应式依赖
computed: {
  now: function () {
    return Date.now()
  }
}

# 方法

当 data 选项中的 msg 发生改变，但 msg 在页面中没有插值渲染，方法就不会重新执行。 当 data 选项中的 msg 发生改变，msg 有在页面中有插值渲染，方法就会重新执行，不管 msg 是否依赖该方法。 每当触发重新渲染时，调用方法将总会再次执行函数。不需要渲染则不会重新执行。

如果你不希望有缓存，请用方法来替代。

<p>{{ now() }}</p>
1
methods: {
  now: function () {
    return Date.now()
  }
}
//html中插入{{now()}}

# 结论：

  1. 用计算属性能实现的问题，用方法一定能实现。而方法甚至还能接受参数，使用起来更灵活。
  2. 唯一区别就是缓存与否的问题。即为什么需要计算属性缓存？

# 性能开销较大

当文本插值表达式过长，逻辑更为复杂时，就会变得十分臃肿，难以阅读和维护时，我们才会选择计算属性来代替过滤器，但恰恰由于计算复杂，导致性能开销较大，可能需要遍历一个巨大的数组并做大量的计算。因此需要缓存。

# 有其他依赖，可能出现多次使用

可能有其他的计算属性依赖于这个计算属性，多次被调用这个计算属性值时不可避免多次执行计算属性的 getter，因此需要缓存。

  1. 想要实时更新肯定是优先选 methods 方法，如果有数组需要遍历且就是用那么 1 次则首选 computed 计算属性。
