# Vuejs 的基本介绍和概述

# Vuejs 是什么

简单小巧、渐进式、功能强大的技术栈。

# Vuejs 的模式

MVVM 是把 MVC 里的 Controll 改成了 ViewModel MVVM = Model + View + ViewModel View <===> ViewModel <===> Model ViewModel 负责监视 View 和 Model 的变化，View 和 Model 的变化会自动更新到 ViewModel 再同步到对方中，实现视图层和数据层的双向绑定 MVVM 模式让我们无需再去关心 DOM 操作的事情，更过的精力放在数据和业务逻辑上去

# el 选项

el是必不可少的选项，指定一个页面中已存在的 DOM 元素，并挂载 Vue 实例。

# data 选项

通过 Vue 实例的data选项，可声明应用内需要双向绑定的数据。

  1. 将会用到的数据预先声明在 data 中
  2. 声明的数据也可以指向一个已有的变量

# vue 实例生命周期

  • 注意：文档中建议 Vue 实例的生命周期钩子不要写成箭头函数。
  • 理由是：箭头函数是和父级上下文绑定在一起的，this 不会是如你所预期的 Vue 实例
  • 经常导致 Uncaught TypeError: Cannot read property of undefined 或 Uncaught TypeError: this.myMethod is not a function 之类的错误。
let app = new Vue({
  el: "",
  data: {},
  beforeCreate: function () {
    alert(
      "Vue实例初始化之后，还没有进行数据观测 (data observer) 和 event/watcher 事件配置。"
    );
  },
  created: function () {
    alert(
      "已完成数据观测 (data observer)，property 和方法的运算，watch/event 事件回调 的配置，但挂载阶段(挂载到DOM上)还没开始，$el property 尚不可用。"
    );
    // 通常把初始化操作放在此周期里
    // 原因一：一次渲染，将统一结算下面三句代码，
    this.xx = false;
    this.xx = true;
    this.xx = false;
    // 页面只能看到一次false的效果
    this.$nextTick(() => {
      // 但是如果我写在下一个周期，或异步操作，将能肉眼看到this.xx切换的效果
      // 原因二：写在这里为以后再次添加其他初始化操作余留一个空间，可以写在beforeMount中
      this.xx = true;
      console.log("我其实发生在beforeMount");
    });
  },
  beforeMount: function () {
    alert(
      "挂载开始之前。 Vue 实例在实例化时没有收到 el 选项，则它处于“未挂载”状态。"
    );
  },
  mounted: function () {
    alert("已经挂载到DOM上，el被$el替换。但不能保证全部el被替换");
    this.$nextTick(() => {
      console.log("我没有下一个周期，所以还是在mounted执行");
    });
  },
  beforeUpdat: function () {
    alert(
      "数据更新后，发生在虚拟 DOM 打补丁之前。这里适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器。"
    );
  },
  updated: function () {
    alert(
      "由于数据更改导致的虚拟 DOM 重新渲染和打补丁，DOM 已经更新,现在可以执行依赖于 DOM 的操作。"
    );
    // 你应该避免在此期间更改状态。如果要相应状态改变，通常最好使用计算属性或 watcher 取而代之。
  },
  activated: function () {
    alert("被 keep-alive 缓存的组件激活。");
  },
  deactivated: function () {
    alert("被 keep-alive 缓存的组件停用时调用。");
  },
  beforeDestroy: function () {
    // 主要作用：解绑一些使用addEventListener监听事件等。
    alert("实例销毁之前。在这一步，实例仍然完全可用。");
  },
  destroyed: function () {
    alert(
      "实例销毁后。被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。"
    );
  },
  errorCaptured: function () {
    alert(
      "捕获一个来自子孙组件的错误。会收到三个参数：错误对象、发生错误的组件实例、一个包含错误来源信息的字符串。可返回 false 以阻止错误继续向上传播。"
    );
  },
});

# 过滤器

Vue 支持在{{}}插值尾部添加|管道符对数据进行过滤，常用于格式化文本，比如字母大写、千位使用逗号分隔符等。 通过给 Vue 实例添加选项 filters 来设置。

{{date|filter1(option)|filter2}}
1
app.$filter1(value, option);
app.$filter2(value);

filters{
  filter1:function(){},
  filter2:function(){},
}

# 指令和事件

指令(Directives)是 Vue 模板中最常用的一项功能，它带有前缀 v-，能帮问哦们快速完成 DOM 操作、循环渲染、显示和隐藏。

指令 用途
v-text 解析文本
v-html 解析 html
v-bind 基本用途是动态更新 HTML 元素上的属性，比如 id、class 等
v-on 绑定事件监听器

# v-text

<!-- app.msg值为字符串 -->
<div v-text="msg"></div>
<!-- 等同于 -->
<div>{{msg}}</div>
