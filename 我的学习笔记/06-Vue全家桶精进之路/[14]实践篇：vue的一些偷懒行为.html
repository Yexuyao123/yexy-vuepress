# 一些特性

# 关于模板语法——计算出最少需要重新渲染的组件

  • 在底层的实现上，Vue 将模板编译成虚拟 DOM 渲染函数。
  • 结合响应系统，Vue 能够智能地计算出最少需要重新渲染多少组件，并把 DOM 操作次数减到最少。
  • 你也可以不用模板，直接写渲染 (render) 函数，使用可选的 JSX 语法。
  • 模板内的表达式非常便利,但是设计它们的初衷就是用于简单运算的，在模板中放入太多的逻辑，会让模板过重且难以维护，此时就可以考虑计算属性和监听器

# 关于 v-if 条件渲染——惰性：条件为假即不渲染

  • 带有v-show 的元素会被渲染并保留在 DOM 中，v-show 只是简单地切换元素的 CSS 属性 display
  • v-show 不支持 template元素(标签)，也不支持 v-else
  • v-if 是“真正”的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。
  • v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。
  • 相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染
  • 一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。
  • 因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

# v-model.lazy 不实时更新

  • v-model.lazy 不实时更新，用户输入完毕并按 enter 键后更新,或者用户鼠标点击别处(即失焦、退出输入状态)时更新

# 一些防偷懒的情况

# 条件渲染指令 v-if + 可复用元素 input

  • v-if、v-else-if、v-else

  • 用法：必须跟着屁股走

  • v-if 的弊端：Vue 在渲染元素时，出于效率考虑，只会渲染有变化的元素，会尽可能地重复利用已有的元素而非重新渲染。

  • 解决方案：加 key 属性使元素变唯一，提供 key 值可以来决定是否复用该元素。

  • 案例

  • 这里 label 元素和 input 标签，当用户切换 v-if 和 v-else 两种状态的时候，vue 并不会重新渲染。

  • 所以当用户输入时，切换状态，由于 input 元素没有重新渲染，输入的内容会保留在输入框中，而不会刷新成当前状态下 input 标签的 placeholder 值。

# 用 key 管理可复用的元素

  • Vue 提供了一种方式来表达“这两个元素是完全独立的，不要复用它们”。
  • 只需添加一个具有唯一值的 key attribute即可。
<!-- 注意，label 元素仍然会被高效地复用，因为它们没有添加 key attribute。 -->
<template v-if="loginType === 'username'">
  <label>Username</label>
  <input placeholder="Enter your username" key="username-input" />
</template>
<template v-else>
  <label>Email</label>
  <input placeholder="Enter your email address" key="email-input" />
</template>

# 路由变化 但页面没有重新渲染

  • 当路由只有查询参数发生变化时，并不会刷新页面上的 view 视图。

# 方案一：添加一个具有唯一值的 key attribute

  • 不推荐，这个方案是自己试出来的
  • 添加 key 属性只能确保重新渲染 DOM，并不能保证能够重新在当前页面重新请求
  • 除非这个 DOM 抽离成单独组件，并配置其独立的请求
  • 当重新渲染组件时，组件被重新调用，写在声明周期中的调用请求方法的函数才会被重新执行。
<template>
  <div id="articleAuthor">
    <main v-if="authorName">
      <div id="content">
        <!-- 对每一个组件添加 key attribute。 -->
        <Article :id="articleId" :key="getSymbol(articleId)"></Article>
        <ArticleReplies
          :id="articleId"
          :key="getSymbol(articleId)"
        ></ArticleReplies>
      </div>
      <div id="sidebar">
        <div class="panel">
          <Author :name="authorName" :key="getSymbol(authorName)"></Author>
        </div>
      </div>
    </main>
  </div>
</template>

<script>
methods: {
  getSymbol(key) {
    return Symbol(key);
  },
},
</script>

# 方案二：使用 watch 监听路由变化（推荐）

<script>
import xxx from "@/components/xxx.vue";

export default {
  name: "ArticleAuthor",
  components: { xxx },
  data() {
    return {
      // 与路由相关的变量
      articleId: this.$route.params.articleId,
      authorName: null
    };
  },
  methods: {
    getArticleData() {
      // console.log(this.id);
      this.$http
        .get("https://cnodejs.org/api/v1/topic/" + this.articleId)
        .then(res => {})
        .catch(function(err) {});
    }
  },
  beforeMount() {
    this.getArticleData();
  }
  watch:{
    // 使用watch监听路由变化
    "$route"(to,from){
      // 路由发生变化就重新请求数据
      this.getArticleData();
    }
  }
};
</script>

# watch 监听函数

  • watch：{ [key: string]: string | Function | Object | Array }
  • 键是需要观察的表达式，
  • 值是对应回调函数。值也可以是方法名，或者包含选项的对象。
  • Vue 实例将会在实例化时调用 $watch()，遍历 watch 对象的每一个 property。
  • 注意，不应该使用箭头函数来定义 watcher 函数 (例如 searchQuery: newValue => this.updateAutocomplete(newValue))。
  • 理由是箭头函数绑定了父级作用域的上下文，所以 this 将不会按照期望指向 Vue 实例，this.updateAutocomplete 将是 undefined。
<script>
var vm = new Vue({
  data: {
    a: 1,
    b: 2,
    c: 3,
    d: 4,
    e: {
      f: {
        g: 5,
      },
    },
  },
  watch: {
    a: function (val, oldVal) {
      console.log("new: %s, old: %s", val, oldVal);
    },
    // 方法名
    b: "someMethod",
    // 该回调会在任何被侦听的对象的 property 改变时被调用，不论其被嵌套多深
    c: {
      handler: function (val, oldVal) {
        /* ... */
      },
      deep: true,
    },
    // 该回调将会在侦听开始之后被立即调用
    d: {
      handler: "someMethod",
      immediate: true,
    },
    // 你可以传入回调数组，它们会被逐一调用
    e: [
      "handle1",
      function handle2(val, oldVal) {
        /* ... */
      },
      {
        handler: function handle3(val, oldVal) {
          /* ... */
        },
        /* ... */
      },
    ],
    // watch vm.e.f's value: {g: 5}
    "e.f": function (val, oldVal) {
      /* ... */
    },
  },
});

vm.a = 2; // => new: 2, old: 1
</script>

# 方案三：调用路由守卫 beforeRouteUpdate 监听路由改变(推荐)

const User = {
  template: "...",
  components: {},
  data: function () {
    return;
  },
  beforeRouteUpdate(to, from, next) {
    // react to route changes...
    // don't forget to call next()
    this.getArticleData();
    next();
  },
};

# 不可用方案

  • 可以注意到的是
  • 编程式导航router.push(location, onComplete?, onAbort?)中也提供了 onComplete 和 onAbort 回调作为第二个和第三个参数。
  • 这两个钩子函数(回调)将会在导航成功完成 (在所有的异步钩子被解析之后) 或终止 (导航到相同的路由、或在当前导航完成之前导航到另一个不同的路由) 的时候进行相应的调用。
  • 但是：如果目的地和当前路由相同，只有参数发生了改变 (比如从一个用户资料到另一个 /users/1 -> /users/2)，你需要使用 beforeRouteUpdate 来响应这个变化 (比如抓取用户信息)。

# 多行操作 Vue 会统一清算

  • 案例
红色蒲公英 2021/04/03 11:40:22
<template>
  <div class="register" class="flex-center" ref="register">
    <Register ref="block"></Register>
    <div v-if="flag" ref="flag"></div>
    <button @click="onClick">点我<button>
  </div>
</template>

<script>
import Register from "@/components/Register.vue";

export default {
  name: "Register",
  components: { Register },
  data() {
    return { flag: false };
  },
  created() {
    // created阶段获取不到ref
    // 还没挂载到dom树上，所以获取不到
    // 但是可以配合this.$nextTick(() => {在下一个周期执行})去完成，created的下一个生命周期是beforeMount
    this.init();
  },
  beforeMount(){
    // 挂载开始之前
  },
  mounted() {
    // 挂载已完成
    // mounted阶段取到dom的方法
    const registerDiv = this.$refs.register;
    // vue获取到组件Register当前实例,实例中的data/methods都可以被访问到
    const RegisterUserName=this.$refs.block.username;
    this.$refs.block.registerCount(); //执行实例的某个方法，注意这里的运行环境是实例内部
  },
  methods: {
    init() {
      this.$nextTick(() => {
        //下一个生命周期里运行
        //this.$refs.flag; //已经被挂载了
      });
    },
    onClick() {
      // this.flag初始值是false，没有挂载渲染
      this.flag = true; //这里将this.flag改为true,下一步马上获取其dom
      this.$refs.flag; //undefined 并不会马上挂载
      this.flag = false;
      this.flag = true; //几步的更改this.flag值vue会进行统一结算，直接到最后一步的结果
      this.$nextTick(() => {
        //到下一个生命周期里运行
        this.$refs.flag; //已经被挂载了，这里就可以被获取到
      });
    },
  },
};
</script>

# Vue 无法探测普通的新增对象 property

<!DOCTYPE html>
<html>
  <head>
    <meta name="description" content="vue" />
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>JS Bin</title>
    <script src="https://cdn.bootcss.com/vue/2.5.16/vue.min.js"></script>
  </head>

  <body>
    <div id="app">
      <ul>
        <!-- 循环遍历数组items，生成listItem -->
        <!-- 数组items初始值是空数组，所以一开始没有生成DOM -->
        <li v-for="(item,index) in items">
          <child> {{item.name}} - {{item.clicked}} </child>
          <button @click="remove(index)">删除</button>
        </li>
      </ul>
      <!-- 点击按钮为数组增加属性值，但是并没有动态渲染出DOM -->
      <button @click="add">add</button>
    </div>
  </body>
</html>

  • 上面代码中有一个 BUG：点击 add 后 this.items 变化了，但是页面却没有变化,请问这是为什么？

  • 由于 JavaScript 的限制，Vue 不能检测到this.items[this.items.length]的变化，所以页面没有重新渲染。

  • 如果改成 this.items.push 就可以解决这个 BUG 了，因为 Vue 在 this.items.push 方法上做了一些手脚

  • 使用 this.$set(this.items, this.items.length, ...) 能解决这个 BUG

  • Array.prototype.$set 弃用 (opens new window)

  • 用 Vue.set 替代 (opens new window)

  • Vue.set( target, propertyName/index, value )

  • 参数：

  • {Object | Array} target

  • {string | number} propertyName/index

  • {any} value

  • 返回值：设置的值。

  • 用法：

  • 向响应式对象中添加一个 property，并确保这个新 property 同样是响应式的，且触发视图更新。

  • 它必须用于向响应式对象上添加新 property，因为 Vue 无法探测普通的新增 property (比如 this.myObject.newProperty = 'hi')

  • 注意对象不能是 Vue 实例，或者 Vue 实例的根数据对象。
