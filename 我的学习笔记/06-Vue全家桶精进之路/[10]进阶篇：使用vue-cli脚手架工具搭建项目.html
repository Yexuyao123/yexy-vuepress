# 自学新版本

# 安装

  • Vue CLI 安装 (opens new window)
  • Vue CLI 包名由 vue-cli 改成了 @vue/cli。 如果已全局安装旧版本的 vue-cli (1.x 或 2.x)，需先 npm uninstall vue-cli -g 卸载它。
  • Node 版本要求: Vue CLI 4.x 需要 Node.js v8.9 或更高版本 (推荐 v10 以上)。
  • npm install -g @vue/cli 全局安装
  • vue 查看所有可用命令，验证安装成功
  • vue -V或vue --version 查看当前版本
  • npm update -g @vue/cli升级全局的 Vue CLI 包
  • upgrade [options] [plugin-name] 升级以@vue/cli-plugin-开头的 Vue CLI 相关模块

# 快速原型开发

  • 快速原型开发 (opens new window)
  • npm install -g @vue/cli-service-global 安装一个全局的扩展
  • 目标：使用 vue serve 和 vue build 命令对单个 _.vue 或 _.js 文件进行快速原型开发。

# vue serve 在开发环境模式下零配置为 .js 或 .vue 文件启动一个服务器

  • 新建 App.vue 文件
  • 在 App.vue 文件所在目录运行vue serve
  • 默认配置：vue serve 使用了和 vue create 创建的项目相同的默认设置 (webpack、Babel、PostCSS 和 ESLint)。会在当前目录自动推导入口文件(main.js、index.js、App.vue 或 app.vue )。也可进行手动配置。
  • vue serve MyComponent.vue手动指定入口文件

# vue build 在生产环境模式下零配置构建一个 .js 或 .vue 文件

  • vue build MyComponent.vue 将目标文件构建成一个生产环境的包并用来部署

# 创建一个项目

  • vue create my-project 创建一个项目
  • Manually select features 手动选择功能
  • Linter / Formatter 使用 ESlint 和 Prettier 格式化工具
  • lint on save在每次保存的时候格式化
  • In dedicated config files独立的配置文档

# 开启服务器

npm run serve

# 文档解读

# component 组件 template

  • vue 对象中可以通过 props 接收来自 import 该组件的父组件的数据
  • export vue 对象 将 component 组件抛出去
  • 以上两步可以通过 ES6 结构合并成一步

# view 组件 template

  • view 组件 template 绑定要传递给子组件的数据
  • import component 组件 （子组件）
  • 在 vue 对象中注册 component 组件 （组件注册到 vue 对象中）
  • 【new Vue（vue 对象）实例化 后将实例 export 抛出，也可先将 vue 对象抛出，最后在入口文件一步实例】
  • vue 对象 将 view 组件抛出去

# router 中 js 文档路由控制

  • import view 组件 (view 组件引入）
  • routes 对象的 routes 属性中注册路由
  • {path:路径，name:命名，component:view 组件} (view 组件注册）
  • new VueRouter（routes 对象）实例化 后将实例 export 抛出

# 入口 main.js 文件

  • import router 路由
  • vue 对象 注册 router 路由
  • new Vue（vue 对象）实例化

# 入口 App.vue 文件

  • <router-link to="routes对象注册的path">点我切换routes对象注册的view组件</router-link>
  • <router-view />设置 view 视图加载位置
