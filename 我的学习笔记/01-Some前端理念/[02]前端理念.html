# 语义化

  • 语义化，指对文本内容的结构化（内容语义化），选择合乎语义的标签（代码语义化）

  • 便于开发者阅读，维护和写出更优雅的代码的同时，让浏览器的爬虫和辅助技术更好的解析。

  • header 标签 定义文档的介绍信息；标题、logo、slogan、目录部分、搜索框、nav 或相关的 logo；一个页面中 header 标签的个数没有限制，可以为每个内容块添加一个 header；

  • nav 标签 定义导航栏，链接等；nav 一般和 u、li 配合做导航栏；

  • main 标签 定义主要内容；main 标签在一份文档中是唯一的，其后代元素常常包括 article 标签；

  • article 标签 定义文档中可以脱离其他部分，一份独立的内容，通常带有标题，当 article 内嵌 article 时，里外层的内容应该是相关的，比如一篇微博和它的评论；

  • section 标签 与 article 的差别在于，它是整体的一部分，或者是文章的一节，一般来说 section 也会带有标题；

  • aside 标签 侧边栏（与 article 并列存在）或者嵌入内容（在 article 内），通常认为是独立拆分出来而不受整体影响的一部分，作为主要内容的附属信息，如索引，词条列表，或者页面及站点的附属信息，如广告，作者资料介绍等；

  • footer 标签 页脚，通常包含作者、版权信息或者相关链接等；

  • 最开始是 PHP 后端写 HTML，不会 CSS，于是就用 table 来布局。table 使用展示表格的。严重违反了 HTML 语义化。

  • 后来有了专门的写 CSS 的前端，他们会使用div+css 布局，主要是用 float 和绝对定位布局。稍微符合了 HTML 语义化。

  • 再后来，前端专业化，知道 HTML 的各个标签的用法，于是会使用恰当的标签来展示内容，而不是傻傻的全用 div，会尽量使用h1、ul、p、main、header 等标签

# meta viewport

<meta
  name="viewport"
  content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0"
/>
1
2
3
4

  • 手机浏览器是把页面放在一个虚拟的**窗口(viewport)**中，通常这个虚拟的“窗口”（viewport）比屏幕宽，这样就不用把每个网页挤到很小的窗口中（这样会破坏没有针对手机浏览器优化的网页的布局），用户可以通过平移和缩放来看网页的不同部分。移动版的 Safari 浏览器最新引进了 viewport 这个 meta tag，让网页开发者来控制 viewport 的大小和缩放，其他手机浏览器也基本支持。

  • 一开始，所有页面都是给 PC 准备的，乔布斯推出 iPhone 3GS，页面是不适应手机屏幕的，所以乔布斯的工程师想了一个办法，默认把手机模拟成 980px，页面缩小。后来，智能手机普及，这个功能在部分网站不需要了，所以我们就用 meta:vp 让手机不要缩小我的网页。

  • initial-scale：初始缩放比例，也即是当页面第一次 load 的时候缩放比例

  • maximum-scale：允许用户缩放到的最大比例

  • minimum-scale：允许用户缩放到的最小比例

  • user-scalable：用户是否可以手动缩放

# CSS reset 和 normalize.css 有什么区别

  • reset 重置，即之前的样式我不要
  • 例如a{color:red;}抛弃了浏览器的默认样式，覆盖上/重置了新的样式
  • normalize 标准化，让所有浏览器的标签都跟标准规定的默认样式一致
  • 例如，8.0.0 版本 normalize.css，html 标签行高 1.15，强调了样式与规范一致，使得各个浏览器样式得到统一

# 选择器如何确定

# 这段代码里的 this 指的是什么

  • 直接使用一个函数：fn()，这里面的 this 就是 window
  • fn()是 strict mode（严格模式）,这里面的 this 就是 undefined
  • a.b.c.fn()，这里面的 this 就是 a.b.c
  • new Function(),这里面的 this 就是 新生成的实例
  • （）=>console.log(this),箭头函数里面的 this 就是外面的 this
  • 事件监听里面的监听函数中的 this 指的是该事件

# 如何实现深拷贝

  • 1.JSON 来深拷贝JSON.parse(JSON.stringify(obj))
  • 缺点：JSON 不支持函数、引用、undifined、正则 RegExp、Date 等
  • 2.使用递归拷贝：要点 1(区分是数组、函数还是对象)，要点 2(递归)
function clone(object) {
  // 区分类型，根据类型分别做拷贝
  var object2;
  // 判断是否普通类型
  if (!object instanceof Object) {
    return object;
  }
  // 判断是否复杂类型
  if (object instanceof Array) {
    object2 = [];
  } else if (object instanceof Function) {
    object2 = eval(object.toString());
  } else if (object instanceof Object) {
    object2 = {};
  }
  for (let key in object) {
    object2[key] = clone(object[key]);
  }
  return object2;
}

function Animal() {
  // 动物有肉体
  this.body = "肉体";
}
// 动物有移动的行为
Animal.prototype.move = function () {};

function Human(name) {
  // 人类是动物
  // 自身属性的继承，人类有肉体
  Animal.apply(this, arguments);

// 原型属性的继承
Human.prototype.__proto__ = Animal.prototype; //原理，但这个操作时违法的，使用下面的方法实现

const f = function () {}; //声明一个空函数
f.prototype = Animal.prototype;
Human.prototype = new f();

//上面实现了Human继承了Animal

Human.prototype.useTools = function () {};

# 事件委托是什么？有什么好处？

  1. 假设父元素有 4 个儿子，不监听 4 个儿子，而是监听父元素，看触发事件的元素是哪个儿子，这就是事件委托。
  2. 好处：可以监听还没有出生的子元素（动态生成的元素），且省监听器(不是监听函数，监听器是浏览器的一个概念)
  3. 关键点：监听的 dom(选择器)和触发监听的 dom(e.cuttentTarget.tagName)是否一致
//DOM结构设定：ul>li*5>span
function listen(container, selector, eventType, fn) {
  //默认冒泡，对ul监听，子元素也能监听到
  container.addEventListener(eventType, (e) => {
    //查看用户点击的元素：e.currentTarget
    let cuttentTarget = e.currentTarget;
    while (currentTarget) {
      if (cuttentTarget.tagName.toLowerCase() === selector) {
        //点击元素的标签名===selector(li) 说明点击的是li标签
      } else {
        //点击元素的标签名!==selector(li) 说明点击的不是li标签，而是li的儿子span
        //就获取点击元素span的父元素li
        cuttentTarget = cuttentTarget.parentNode;
      }
      fn.call(e);
    }
  });
}

# 301 和 302 的区别是什么

# Cookie 和 Session 分别是什么

  • Cookie
  1. HTTP 响应通过 Set-Cookie 设置 Cookie
  2. 浏览器访问指定域名必须带上 Cookie 作为 Request Header
  3. Cookie 一般用来记录用户信息
  • Session
  1. Session 是服务器端的内存（数据）
  2. Session 一般通过在 Cookie 里记录 SessionID 实现
  3. SessionID 一般是随机数

# LocalStorage 和 Cookie 区别是什么

  • Cookie 会随请求被发送到服务器上，而 LocalStorage 不会
  • Cookie 大小一般 4k 以内，LocalStorage 大小一般 5Mb 左右

# GET 和 POST 的区别是什么

  1. 参数：GET 的参数放在 url 的查询参数里面，POST 的参数放在请求消息体里
  2. 安全性：都不安全，GET 没有 POST 安全
  3. 长度：GET 的参数（url 查询参数）有长度限制（一般是 1024 个字符），POST 的参数，没有长度限制（扯淡，一般是 4-10Mb）
  4. 包：GET 请求只需要发一个包，POST 请求需要发两个以上的包（POST 有消息体）（扯淡，GET 也可以有消息体）
  5. GET 用来读取数据，POST 用来存写数据，POST 不幂等（幂等：即不管发多少次请求，结果都一样）
  6. 扯淡，两者其实没有区别

# 安全：什么是 CSRF 攻击？如何预防？

  • 什么是 CSRF 攻击（百度）
  1. 用户在 qq.com 登录
  2. 用户切换到 hacker.com(恶意网站)
  3. hacker.com 发送一个 qq.com/friends 请求，请求用户的好友信息
  4. 或者 hacker.com 发送一个 qq.com/addfriends 请求，让用户添加 hacker 为好友
  5. 用户的好友信息就被 hacker.com 偷走了，或者在不知不觉中添加了 hacker 为好友了
  6. 虽然有跨域，但是跨域并不阻止你发送请求，跨域只阻止你接受响应
  7. 也就是说用户没有想要发送这个请求，但是 hacker 伪造了用户发送请求的假象
  • 如何避免;
  1. 检查 referer,qq.com 可以拒绝来自 hacker.com 的请求
  2. 使用 csrf_token 来解决
