# 1.命令行工具基础命令

# .移

# 切换当前目录

  • cd 文件夹或路径 切换文件夹 or 进入一个文件夹
  • cd C:/ 去 C 盘
  • cd .. 返回上一个文件夹
  • cd //cd空../.. 回到根目录(即此电脑)
  • cd /目录路径 到目录路径所指这个文件夹
  • cd ~ 回到用户主目录(~环境变量里找)
  • cd ~/Desktop/ 去桌面

# 移动文件

  • cp a b 将 a 复制并命名为 b
  • mv a b 将 a 路径移动到 b 路径

# .增·创建

  • mkdir demo-11 创建文件夹并命名为 demo-11(makedirectory)
  • touch index.html 创建文件并命名为 index.html

# .删·删除

  • rm 文件路径 删除文件/文件夹
  • rm -rf * 删除当前目录下的所有文件,这个命令很危险，应避免使用
  • rm [选项] 文件名 删除格式

# [选项]

  • -f/--force 忽略不存在的文件，强制删除，不给出提示
  • -r/-R --recursive 指示 rm 将参数中列出的全部目录和子目录均递归地删除
  • -i --interactive 进行交互式删除,逐个输入“y”确认要删除的文件
  • -v --verbose 详细显示进行的步骤

# .改·编辑

  • vi index.html 编辑文件
  • vim index.html 编辑文件(强化版本)
  • echo '1' > 文件路径把(内容:1)写进文件
  • curl -L https://www.baidu.com > baidu.html 下载链接地址里的文件并存到 baidu.html 里面

# .查·查询

  • ls 查看当前文件夹所有文件
  • ls -l 路径 查看文件夹文件,显示更多信息
  • pwd 查看当前位置
  • cat 文件路径 查看文件内容

# .其他

  • clear 清屏
  • ctrl+d 退出 node
  • ctrl+c 中断
  • shift+ins 粘贴

# 2.npm 常用命令

# .基础

  • npm login 登录
  • npm adduser 添加用户
  • npm config set registry https://registry.npm.taobao.org 添加镜像
  • npm run 执行在 package.json 的 scripts 中定义的脚本命令

# .查询

  • npm -v 查看当前 npm 版本，检查是否安装成功
  • npm root 查看当前包的安装路径
  • npm root -g 查看全局的包的安装路径
  • npm help 查看帮助命令
  • npm ls/npm list 查看已安装的模块

# .包/模块

# 发布

  • npm init -y 初始化(创建一个 package.json 文件，包括名称、版本、作者等信息)
  • npm search packageName 发布一个 npm 包的时候，需要检验某个包名是否已存在
  • npm publish 发布包
  • npm -f unpublish dollarphpajax@\* 撤销发布

# 安装

  • npm install 根据 package.json 文件安装
  • npm install 包名 本地安装，在 npm 运行目录下安装
  • npm install 包名 -g 全局安装 npm -g install 包名/npm install -g 包名均可
  • 注意：-g表示 global 表示在全局变量中进行操作，可以放在 install 和包名前后任意位置空格隔开
  • npm install 包名@1.x 安装指定版本

# 卸载

  • npm uninstall 包名/模块名 卸载包/node 模块

# 更新

  • npm -g update 包名 全局更新
  • npm update 包名/模块名 本地更新包/node 模块
  • npm rebuild 包名/模块名 更改包内容后进行重建

# .杂项

  • npm config delete registry 恢复镜像
  • npm config edit 手动删除
  • npm link 在本地安装好的模块中，运行(添加本地模块到全局链接)
  • npm link 模块名 在要使用的本地目录中，运行(从全局中获取链接)
  • npm help folders 查看 npm 使用的所有文件夹(跳转 cli 命令)
  • npm view express 查看 node 模块的 package.json 文件夹
  • npm view moudleName dependencies 查看包的依赖关系
  • npm view moduleName repository.url 查看包的源文件地址
  • npm view moduleName engines 查看包所依赖的 node 的版本
  • npm outdated 检查包是否已经过时，列出所有已经过时的包以便及时进行包的更新
  • npm help json 访问 npm 的 son 文件夹的方法,以默认的方式打开网页，可改默认打开程序
