# 安装

# 添加 SSH key

  • 进入 github 用户设置页面 (opens new window)
  • 点「delete」按钮可以删除已有的 key 。
  • 点击 New SSH key，你需要输入 Title 和 Key，但是你现在没有 key，往下看
  • 打开 Git Bash
  • 运行 rm -rf ~/.ssh/*可以把电脑中现有的 ssh key 都删掉，这句命令行如果你多打一个空格，可能就要重装系统了，建议复制运行。
  • 运行 ssh-keygen -t rsa -b 4096 -C "你的邮箱"，注意填写你的邮箱！按回车三次，回车直至完成。
  • 运行 cat ~/.ssh/id_rsa.pub，得到 ssh key 并复制
  • 回到上面第 3 步的页面，输入 Title，粘贴 ssh key 后点击 Add SSH key。
  • 回到 Git Bash 运行 ssh -T git@github.com，你可能会看到……提示：输入 yes 回车……
  • 然后如果你看到 Permission denied (publickey). 就说明你失败了，请回到第 1 步重来；
  • 如果你看到 Hi FrankFang! You've successfully authenticated, but GitHub does not provide shell access. 就说明你成功了！
  • 一台电脑只需要一个 SSH key
  • 一个 SSH key 可以访问你的所有仓库，即使你有 1000000 个仓库，都没问题
  • 如果你新买了电脑，就在新电脑上重新生成一个 SSH key，把这个 key 也上传到 GitHub，它可以和之前的 key 共存在 GitHub 上
  • 如果你把 key 从电脑上删除了，重新生成一个 key 即可，替换之前的 key
  • 至于一台电脑多个 git 账户，就要涉及账户管理，和上传时账户切换了

# git 五行配置命令

Git Bash 命令行运行这五句话

git config --global user.name xxxxxx
git config --global user.email yyyyyy
git config --global push.default simple
git config --global core.quotepath false
git config --global core.editor "vim"

# git config --global user.name xxxxxx

把 xxxxxx 替换成你的英文名字随便什么都行，设置用户名

# git config --global user.email yyyyyy

把 yyyyyy 替换成你的邮箱跟 github 一致或者不一致也行，设置用户邮箱

# git config --global push.default simple

新装/更新了 Git，在执行 git push 时看到如下消息：

warning: push.default is unset; its implicit value is changing in
Git 2.0 from 'matching' to 'simple'. To squelch this message
and maintain the current behavior after the default changes, use:
  git config --global push.default matching
To squelch this message and adopt the new behavior now, use:
  git config --global push.default simple

解决方案：修改全局配置，使之不会每次 push 的时候都进行提示。

  • git config --global push.default matching matching 参数是 Git 1.x 的默认行为，其意是如果你执行 git push 但没有指定分支，它将 push 所有你本地的分支到远程仓库中对应匹配的分支。
  • git config --global push.default simple 而 Git 2.x 默认的是 simple，意味着执行 git push 没有指定分支时，只有当前分支会被 push 到你使用 git pull 获取的代码。

# git config --global core.quotepath false

处理中文文件名

GIT 乱码问题：有编码的原因，也有跨平台的原因。主要原因是 Windows 系统中的 Git 对中文文件名采用不同的编码保存所致。 Windows 系统中使用的 msysGit，采用的是系统编码来保存文件名；而 Cygwin 中的 Git 默认采用 UTF-8 编码来保存文件名。如果两个软件同时对一个版本库进行操作，且都认为对方是使用自己使用的编码来保存文件，就会导致文件名编码混乱，无法识别。 摘自：GIT 乱码解决方案汇总 (opens new window)

  • 没有设置的情况下使用 git bash 可能出现如下问题：
  • fatal: unable to read config file / could not lock config file
  • 使用 PowerShell 可以避免该问题
  • 或者设置git config --global core.quotepath false

# git config --global core.editor "vim"

  • 指定的文本编辑器：使用 vim 编辑提交信息。
  • 核心编辑器/默认编辑器采用 vim 编辑器
  • 当用 git commit 命令没有接-m "说明" 的时候,git 会尝试打开一个编辑器让你输入更改说明,这个时候就会打开 vim 编辑器。
  • 如果将这条命令换成 git config --global core.editor gedit，使用 git commit 时会弹出 geditor 编辑器让我们输入更改说明。

# 查看 git 配置

  • 用 git config 命令查看配置文件
  • 命令参数 –list, 简写 -l
  • 格式：git config (–local/–global/–system) -l
目标 命令 路径
查看版本库级的 config git config --local -l .git\config
查看全局级的 config git config --global -l C:\Users\XXX 用户名\ .gitconfig
查看系统级的 config git config --system -l C:\Program Files\Git\etc\gitconfig
查看当前生效的配置 git config -l，这个时候会显示最终三个配置文件计算后的配置信息 -

# git 常用命令

  • git add .一键添加到暂存区
  • git commit -m"本次提交的说明"将暂存区的文件提交到本地仓库
  • git push本地仓库推送到 git远程仓库
  • git status -sb查看当前状态

# git 分支

# 创建与切换

  • 创建分支：git branch mybranch
  • 切换分支：git checkout mybranch
  • 创建并切换分支：git checkout -b mybranch
  • 实践：
  • git checkout main切换到 main 分支
  • 未找到分支：pathspec main did not match any file(s) known to git
  • 切换成功：Switched to branch 'master',your branch is up to date with 'origin/main'

# 查看分支

  • git branch -a查看远程分支
  • git branch查看本地分支

# 将 xxx 分支合并到 master 分支

# 1.切换到 xxx 分支

git checkout xxx

# 2.拉取 xxx 分支的内容

  • git pull从远程拉取最新版本到本地自动合并 merge git pull origin xxx
  • git fetch从远程获取最新版本到本地不会自动合并 merge git fetch origin xxx
  • 使用git fetch更安全检查确认无误再手动合并到 xxx 中git merge orgin/xxx
  • git pull与git fetch + git merge相同
git pull origin xxx
等同于
git fetch origin
git merge origin/xxx

# 3.进行修改开发，开发完提交保存到远程分支

  • git add .
  • git commit -m "commit infomation"
  • git push origin xxx
  • 这里只是完成了修改与备份

# 4.切换回 master 分支

  • git checkout xxx

# 5.把本地 xxx 分支合并到当前分支

  • git merge xxx将 xxx 分支合并到 master 上
  • Already up to date.

# 6.把本地合并后的 master 同步到远程

  • git add .
  • git commit -m "commit infomation"
  • git push origin master
  • 这里就完成了将远程的 xxx 分支合并到本地的 master 分支，并上传到远程的 master 分支

# 关于 main

  • 2020 年 10 月 1 日后，Github 会将所有新建的仓库的默认分支从 master 修改为 main
  • Github 最近将 master 改为 main，网上有种说法。。。。目的是为了消除对奴隶制的不必要引用，并用更具包容性的术语取代它们，从而摆脱令人不快的“主人”这个词。
  • 可以在仓库的 setting 中修改默认分支
