前言：JSONP(JSON with Padding) 是 JSON 的一种"使用模式"，可以让网页从别的域名（网站）那获取资料，即跨域读取数据。JSONP 的核心原理就是目标页面回调本地页面的方法,并带入参数。

# 1.同源策略

为什么我们从不同的域（网站）访问数据需要一个特殊的技术( JSONP )呢？这是因为同源策略。 同源策略，它是由 Netscape 提出的一个著名的安全策略，现在所有支持 JavaScript 的浏览器都会使用这个策略。

  • 浏览器同源政策及其规避方法 —— 阮一峰 (opens new window)

摘要：

A网页设置的Cookie，B网页不能打开，除非这两个网页"同源"。
所谓"同源"指的是"三个相同"：
> 协议相同
> 域名相同
> 端口相同
如果非同源，共有三种行为受到限制：
> Cookie、LocalStorage 和 IndexDB 无法读取。
> DOM 无法获得。
> AJAX 请求不能发送。

# 服务器搭建 node.js

# 1、常规

  • require("fs") 引入 node.js 的"fs"模块
  • fs.readFileSync("文件路径", "utf-8") 用 utf-8 读取路径对应文件的内容
  • fs.writeFileSync("文件路径", 变量) 将变量的值写入路径对应文件的内容
  • 变量1.replace("字符串", 变量2) 将变量 1 中的"字符串"替换为变量 2 的值

# 2、request 请求的 API

  • request.method GET/POST 等

  • request.url 请求路径

  • url.parse() 可以将一个完整的 URL 地址分为：host、port、pathname、path、query 等

  • URL.path：HTTP 路径(含查询参数)

  • URL.query：查询参数(Url.query)

  • URL.pathname：路径(不含查询参数)

# 3、response 响应的 API

  • response.statusCode 状态码

  • response.setHeader("Content-Type", "text/html;charset=utf-8")

    • response.setHeader：设置请求头
    • ("key", "value")：响应第 2 部分 key：value
    • "value1;value2"：分号隔开

  • response.write("XXXXXX") 响应第 4 部分:要下载的内容

  • response.end() 结束响应

# ① 最土办法：用.split("=")处理字符串
// 例子场景是支付现金
if (path === "/pay" && method.toUpperCase() === "POST") {
  request.on("data", (numberString) => {
    const numberStr = numberString.toString("utf-8");
    const numberArr = numberStr.split("&");
    const number1Arr = numberArr[0];
    const number2Arr = numberArr[1];
    const number1 = number1Arr.split("=")[1];
    const number2 = number2Arr.split("=")[1];

    response.statusCode = 200;
    const amount = fs.readFileSync("./db", "utf-8");
    const newAmount = amount - number1 - number2;
    fs.writeFileSync("./db", newAmount);
    response.write("success");
    response.end();
  });
}

# 3.通过 JSONP 前后端分离

  • 避免由后端直接去操纵 DOM
  • 对于跨域请求：A 网站的后端不可能对 B 网站的 HTML 熟知
  • 所以更应该前后端分离（解耦）
// main.js 中声明了XXX函数
window.XXX = function (result) {
  alert("server.js(服务器node.js)执行了XXX");
  alert("我得到的结果是" + result);
  if (result === "success") {
    alert("付款成功");
  } else {
    alert("付款失败");
  }
};

// server.js 中响应调用XXX函数并传参
if (path === "/pay" && method.toUpperCase() === "GET") {
  response.statusCode = 200;
  response.setHeader("Content-Type", "application/javascript");
  response.write("XXX.call(undefined,'success')");
  response.end();
}

# JSONP(JSON with Padding)

// node.js 中
response.write("XXX.call(undefined,'JSON格式数据')");
1
2

  • "XXX.call(undefined, + 'JSON 格式数据' + )"

  • padding-left + JSON + padding-right

  • JSONP(JSON with Padding) 是 JSON 的一种"使用模式"，可以让网页从别的域名（网站）那获取资料，即跨域读取数据。

  • JSONP 的核心原理就是目标页面回调本地页面的方法,并带入参数。

  • script 标签不受域名限制，但是 AJAX 受到域名限制。

# JSONP 总结

  • 请求方：A.com 的前端程序员（浏览器）
  • 响应方：B.com 的后端程序员（服务器）
  1. 请求方创建 script 标签，src 指向响应方，同时传递 1 个查询参数？callbackName=XXX，XXX 为 callback 函数的函数名。
  2. 响应方根据查询参数 callbackName 调用 callback 函数，构造形如下方形式的响应：
    • XXX.call(undefined,"请求方请求的数据")
    • XXX("请求方请求的数据")
  3. 浏览器接收到响应，就会执行 callback 函数
  4. 请求方获得其想要的数据

# 提问

为什么 JSONP 不支持 POST 请求？

  • 答：
  • 因为 JSONP 是通过动态创建 script 标签实现的。
  • 动态创建 script 标签时只能用 GET 请求，不能用 POST 请求。
