# window.localStorage 常用 API

# 存值 localStorage.setItem()

  • localStorage.setItem(key,value) 即 Storage[key] = value
  • key 和 value 都必须是字符串，如果不是，会先 .toString()转为字符串再存入 Storage 对象中。
  • 注意任何对象.toString(),都会得到"[object object]"
  • 解决方案：JSON.stringify(obj)转为 JSON 字符串
localStorage.setItem("小明",JSON.stringify({
  name："小明"，
  age:18
}))

# LocalStorage 特点

所有页面公用一个 C 盘文件，localStorage 是一种持久花存储

  1. localStorage 跟 HTTP 无关
  2. HTTP 不会带上 LocalStorage 的值
  3. 只有相同域名的页面才能互相读取 localStorage（没有同源那么严格）
  4. 每个域名 localStorage 最大存储量为 5Mb 左右（每个浏览器不一样）
  5. 常用场景：记录有没有提示过用户（没有用的信息，不能记录密码）
  6. localStorage 永久有效，除非用户清理缓存

# 用户清理 LocalStorage 缓存

  • 按住 Ctrl+Shift+Delete 打开设置清除浏览记录；
  • 高级 → Cookie 及其他网站数据；
  • 其他网站数据中就包含了 localStorage。

# 补充 SessionStorage（会话存储）

  1. 1、2、3、4 同 LocalStorage；
  2. SessionStorage 在用户关闭页面（会话结束）后就失效。 准确说是 Session 会话结束后失效
  3. 使用 API 也一样。
sessionStorage.getItem();
sessionStorage.setItem();
sessionStorage.removeItem();
sessionStorage.clear(); // 清空 Storage 对象

# Cookie/Session/LocalStorage/SessionStorage 的联系

  1. Session 是基于 Cookie 实现的，Cookie 是 Session 的基石；
  2. Cookie 每次请求时会带给服务器，localStorage 与 HTTP 无关不会带出去；
  3. Cookie 4k 左右，localStorage 4M 左右；
  4. Cookie 默认在用户关闭页面后失效，但后台代码(服务器)可以设置 Cookie 过期失效时间；
  5. LocalStorage 永久有效，SessionStorage 在用户关闭页面后就失效；
