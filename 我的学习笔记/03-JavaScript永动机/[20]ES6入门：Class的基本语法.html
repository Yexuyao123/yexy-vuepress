# 类的由来

  • ES6 的 class 绝大部分功能，ES5 都可以做到，新的 class 写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。
  • 比较新旧语法：
// ES5
function Point(x, y) {
  this.x = x;
  this.y = y;
}
Point.prototype.toString = function () {
  return "(" + this.x + ", " + this.y + ")";
};
var p = new Point(1, 2);

// ES6
class Point {
  // constructor()方法就是构造方法，而this关键字则代表实例对象。
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
  // toString()方法。
  // 注意，定义toString()方法时，前面无需加function关键字，直接把函数定义放进去就可以。
  // 方法之间不需要逗号分隔，加了会报错。
  toString() {
    return "(" + this.x + ", " + this.y + ")";
  }
}
// 类的数据类型就是函数，类本身就指向构造函数。
typeof Point; // "function"
Point === Point.prototype.constructor; // true
// 使用
const P = new Point();

# 类的方法

  • constructor()方法就是构造方法，而 this 关键字则代表实例对象。
  • 方法之间不需要逗号分隔，加了会报错。
  • 使用：直接对类使用 new 命令，跟构造函数的用法完全一致。
  • 类的所有方法都定义在类的 prototype 属性上面。

# constructor 方法

  • constructor()方法是类的默认方法，通过 new 命令生成对象实例时自动调用该方法。
  • 类必须有 constructor()方法，若没有显式定义，会默认添加一个空的 constructor()方法。
  • 类必须使用 new 调用，普通构造函数不用 new 也可以执行，类不用 new 会报错。
  • constructor()方法中 this 关键字则代表实例对象。
  • constructor()方法默认返回实例对象（即 this），完全可以指定返回另外一个对象。
class Foo {
  constructor() {
    return Object.create(null);
  }
}

# 类的实例

# 实例属性定义位置

  • 实例的属性除非显式定义在其本身（即定义在 this 对象上），否则都是定义在原型上（即定义在 class 上）。
//定义类
class Point {
  constructor(x, y) {
    // 显式定义在其本身 this 对象上
    this.x = x;
    this.y = y;
  }
  // 定义在原型(即 class)上
  toString() {
    return "(" + this.x + ", " + this.y + ")";
  }
}
var point = new Point(2, 3);

point.toString(); // (2, 3)
point.hasOwnProperty("x"); // true
point.hasOwnProperty("y"); // true
point.hasOwnProperty("toString"); // false
point.__proto__.hasOwnProperty("toString"); // true

# 类的所有实例共享一个原型对象

// p1/p2/p3由同一个实例Point创建，所以原型链相同
var p1 = new Point(2, 3);
var p2 = new Point(3, 2);
// 改变p1的原型链，等同于改变p2/p3的原型链
// 所以不推荐这样修改类实例的原型链
p1.__proto__.printName = function () {
  return "Oops";
};

p1.printName(); // "Oops"
p2.printName(); // "Oops"
var p3 = new Point(4, 2);
p3.printName(); // "Oops"

# 属性的赋值与读取

class MyClass {
  constructor() {
    // ...
  }
  get prop() {
    return "getter";
  }
  set prop(value) {
    console.log("setter: " + value);
  }
}

let inst = new MyClass();
inst.prop = 123;
// setter: 123
inst.prop;
// 'getter'

  }
})("张三");
person.sayName(); // "张三"

# Generator 方法

class Foo {
  constructor(...args) {
    this.args = args;
  }
  *[Symbol.iterator]() {
    for (let arg of this.args) {
      yield arg;
    }
  }
}

for (let x of new Foo("hello", "world")) {
  console.log(x);
}
// hello
// world

# 静态属性与静态方法 VS 动态

class A {
  //class 是语法糖，本质是通过prototype添加属性和方法
  //static关键字表示静态的，这个属性或方法属于这个class，而不是实例
  //不加static就是动态的，属于实例
  number = 0;
  static count = 0;
  static show() {
    console.log(this);
  }
  static add() {
    A.count++;
  }
  constructor(a: number) {
    // 这是个动态方法，可以通过A.count访问到静态属性，并修改其值
    this.number = a;
    A.count = A.count + a;
  }
}

A.show(); //可以直接运行这个静态方法
let a = new A(21);
a.number; //21 //这个number属性是动态属性，属于实例本身，不同的实例拥有自己的number属性
// a.show访问不到，静态的属性属于class的
console.log(A.count); //21
let b = new A(321);
b.number; //321
console.log(A.count); //21+321 = 342
//谁调用了这个class就会永久在static属性中留下痕迹
// 动态的属性属于类与所有实例公用的
A.add();
A.count; //343

# 私有属性与私有方法

  • 私有方法和私有属性，是只能在类的内部访问的方法和属性，外部不能访问。
  • 这是常见需求，有利于代码的封装，但 ES6 不提供，只能通过变通方法模拟实现。

# 方案 1：命名上加以区别

class Widget {
  // 公有方法
  foo(baz) {
    this._bar(baz);
  }
  // 私有方法
  _bar(baz) {
    return (this.snaf = baz);
  }
}
// 但，这种命名是不保险的，在类的外部，还是可以调用到这个方法。
// 只是用来告诉开发者这是私有的而已。

# 方案 2：将私有方法移出类

  • 类内部的所有方法都是对外可见的。
class Widget {
  foo(baz) {
    // 类的方法foo调用了外部函数bar
    // 并通过.call修改this指向为当前的this
    bar.call(this, baz);
  }
  // ...
}

function bar(baz) {
  // 外部声明的方法对this的snaf属性进行赋值
  return (this.snaf = baz);
}

# 方案 3：利用 Symbol 值的唯一性

const bar = Symbol("bar");
const snaf = Symbol("snaf");

export default class myClass {
  // 公有方法
  foo(baz) {
    this[bar](baz);
  }
  // 私有方法
  [bar](baz) {
    return (this[snaf] = baz);
  }
  // ...
}

# 方案 4: 在属性/方法名前使用#表示（推荐）

class IncreasingCounter {
  #count = 0;
  get value() {
    console.log("Getting the current value!");
    return this.#count;
  }
  increment() {
    this.#count++;
  }
}

const counter = new IncreasingCounter();
counter.#count; // 报错
counter.#count = 42; // 报错

# 方案 5：用 TypeScript 啊

  • JavaScript 是一门动态语言，没有类型声明。那就使用有类型声明的 TypeScript
  • 但是，TS 的这个 private 和 public 的声明是无效的，只是用于约束开发者。。。
export class VueAxios {
  private axiosInstance: AxiosInstance;
  private readonly options: AxiosRequestConfig;
  axiosCanceler: AxiosCanceler;
  //构造函数
  constructor(
    baseURL: string,
    options: AxiosRequestConfig = {
      headers: {
        "Content-Type": "application/json",
      },
      withCredentials: true,
      timeout: 30000,
    }
  ) {
    this.options = options;
    this.axiosInstance = Axios.create({ baseURL, ...options });
    this.axiosCanceler = new AxiosCanceler();
    this.setupAxiosHooks();
  }
}
